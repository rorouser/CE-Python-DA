Capítulo 5

Introducción a las Bases de Datos

Aunque Pandas proporciona capacidades avanzadas para análisis de datos, en contextos profesionales los datos suelen residir en bases de datos. La interacción con bases de datos es una competencia fundamental para el desarrollo de aplicaciones web, análisis empresarial y pipelines de ciencia de datos.

Una base de datos es una colección organizada de datos con capacidades de acceso, gestión y actualización eficientes. Independientemente de la arquitectura inicial de un proyecto, los flujos de datos frecuentemente interactúan con una o más bases de datos.

Ventajas de las Bases de Datos

Las bases de datos proporcionan múltiples ventajas respecto al almacenamiento en archivos simples:

1. 💾 Persistencia de Datos

Los datos sobreviven después de que tu programa termina de ejecutarse. Puedes cerrar tu aplicación, reiniciar el servidor, y los datos siguen ahí.

2. 🤝 Compartir Datos

Múltiples aplicaciones pueden acceder a los mismos datos simultáneamente. Tu aplicación web, tu script de análisis y tu dashboard pueden todos leer de la misma fuente de verdad.

3. 🔍 Consultas Eficientes

Las bases de datos están optimizadas para buscar, filtrar y agregar datos rápidamente, incluso con millones de registros.

4. ✅ Integridad de Datos

Las bases de datos relacionales imponen restricciones que garantizan que tus datos sean consistentes y válidos.

Bases de Datos Relacionales

Las bases de datos relacionales, también conocidas como bases de datos de filas y columnas, son el tipo más común en uso hoy en día. Proporcionan una forma estructurada de almacenar datos en tablas, similar a hojas de cálculo de Excel pero mucho más potentes.

En una base de datos relacional:

  • Los datos se organizan en tablas
  • Cada tabla tiene columnas (campos) y filas (registros)
  • Las tablas se relacionan entre sí mediante claves
  • Usas SQL para interactuar con los datos
Base de Datos Características Casos de Uso
SQLite Ligera, sin servidor, archivo único Desarrollo, apps móviles, prototipos
MySQL Popular, código abierto, rápida Aplicaciones web, WordPress, startups
PostgreSQL Avanzada, extensible, robusta Análisis de datos, aplicaciones enterprise

Fundamentos de SQL

SQL (Structured Query Language) es el lenguaje estándar para interactuar con bases de datos relacionales. Aunque Python es el que ejecuta el código, SQL es el que habla con la base de datos.

Anatomía de una Consulta SQL

Las consultas SQL típicamente tienen tres componentes:

Componentes de SQL

SELECT *                    -- 1. OPERACIÓN (qué hacer)
FROM empleados              -- 2. OBJETIVO (sobre qué tabla)
WHERE salario > 40000;      -- 3. CONDICIÓN (qué registros)

Operaciones SQL Comunes

DML: Data Manipulation Language

-- SELECT: Consultar datos
SELECT nombre, edad FROM empleados WHERE edad > 30;

-- INSERT: Insertar datos
INSERT INTO empleados (nombre, edad, salario) 
VALUES ('Ana García', 28, 45000);

-- UPDATE: Actualizar datos
UPDATE empleados 
SET salario = 50000 
WHERE nombre = 'Ana García';

-- DELETE: Eliminar datos
DELETE FROM empleados 
WHERE edad < 20;

DDL: Data Definition Language

-- CREATE: Crear tabla
CREATE TABLE empleados (
    id INTEGER PRIMARY KEY,
    nombre VARCHAR(50),
    edad INTEGER,
    salario REAL
);

-- ALTER: Modificar tabla
ALTER TABLE empleados 
ADD COLUMN departamento VARCHAR(30);

-- DROP: Eliminar tabla
DROP TABLE empleados;

SQLite: Tu Primera Base de Datos

SQLite es perfecto para aprender porque:

  • ✅ Viene incluido con Python (no necesitas instalar nada)
  • ✅ No requiere servidor (es solo un archivo)
  • ✅ Es rápido y ligero
  • ✅ Usa SQL estándar

Crear y Conectar a una Base de Datos

Primera Conexión

import sqlite3

# Conectar a base de datos (se crea si no existe)
conexion = sqlite3.connect('mi_empresa.db')

# Crear cursor (objeto para ejecutar SQL)
cursor = conexion.cursor()

# Crear tabla
cursor.execute('''
    CREATE TABLE IF NOT EXISTS empleados (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        nombre TEXT NOT NULL,
        edad INTEGER,
        salario REAL,
        departamento TEXT
    )
''')

# Guardar cambios
conexion.commit()

print("✅ Base de datos y tabla creadas exitosamente")

# Cerrar conexión
conexion.close()
✅ Base de datos y tabla creadas exitosamente

SQLAlchemy: Abstracción de Bases de Datos

SQLAlchemy es una biblioteca Python que abstrae las diferencias entre diferentes sistemas de bases de datos, facilitando cambiar de SQLite a PostgreSQL o MySQL sin reescribir código.

Instalación

pip install sqlalchemy

Uso con Pandas

import pandas as pd
from sqlalchemy import create_engine

# Crear engine (motor de base de datos)

df = pd.read_sql('SELECT * FROM empleados', engine)

import sqlite3

def insertar_empleado_seguro(nombre, edad, salario, departamento):
    try:
        with sqlite3.connect('mi_empresa.db') as conexion:
            cursor = conexion.cursor()
            cursor.execute('''
                INSERT INTO empleados (nombre, edad, salario, departamento)
                VALUES (?, ?, ?, ?)
            ''', (nombre, edad, salario, departamento))
            conexion.commit()
            print(f"✅ Empleado {nombre} insertado correctamente")
            return True
    except sqlite3.IntegrityError as e:
        print(f"❌ Error de integridad: {e}")
        return False
    except sqlite3.Error as e:
        print(f"❌ Error de base de datos: {e}")
        return False
    except Exception as e:
        print(f"❌ Error inesperado: {e}")
        return False

# Probar

JOINs: Combinando Datos de Múltiples Tablas

Los JOINs son fundamentales en bases de datos relacionales. Permiten combinar datos de diferentes tablas basándose en relaciones entre ellas. Esto refleja el principio de normalización de bases de datos, donde dividimos los datos en múltiples tablas para minimizar redundancia.

Tipos de JOINs

1. INNER JOIN (Intersección)

Devuelve solo las filas donde hay coincidencia en ambas tablas.

import sqlite3
import pandas as pd

# Crear base de datos de ejemplo
with sqlite3.connect('tienda.db') as conn:
    cursor = conn.cursor()
    
    # Tabla de productos
    cursor.execute('''
        CREATE TABLE IF NOT EXISTS productos (
            id INTEGER PRIMARY KEY,
            nombre TEXT,
            precio REAL
        )
    ''')
    
    # Tabla de ventas
    cursor.execute('''
        CREATE TABLE IF NOT EXISTS ventas (
            id INTEGER PRIMARY KEY,
            producto_id INTEGER,
            cantidad INTEGER,
            FOREIGN KEY (producto_id) REFERENCES productos(id)
        )
    ''')
    
    # Insertar datos
    productos = [(1, 'Laptop', 1000), (2, 'Mouse', 20), (3, 'Teclado', 50), (4, 'Monitor', 300)]
    cursor.executemany('INSERT OR IGNORE INTO productos VALUES (?,?,?)', productos)
    
    ventas = [(1, 1, 2), (2, 2, 5), (3, 3, 3)]  # Monitor no tiene ventas
    cursor.executemany('INSERT OR IGNORE INTO ventas VALUES (?,?,?)', ventas)
    conn.commit()

# INNER JOIN: Solo productos con ventas
with sqlite3.connect('tienda.db') as conn:
    query = '''
        SELECT 
            p.nombre,
            p.precio,
            v.cantidad,
            (p.precio * v.cantidad) as total
        FROM productos p
        INNER JOIN ventas v ON p.id = v.producto_id
    '''
    df = pd.read_sql(query, conn)
    
print("📊 INNER JOIN - Solo productos vendidos:")
print(df)
📊 INNER JOIN - Solo productos vendidos: nombre precio cantidad total 0 Laptop 1000.0 2 2000.0 1 Mouse 20.0 5 100.0 2 Teclado 50.0 3 150.0

2. LEFT JOIN (Todas de la izquierda)

Devuelve todas las filas de la tabla izquierda, con coincidencias de la derecha si existen.

import sqlite3
import pandas as pd

# LEFT JOIN: Todos los productos, con o sin ventas
with sqlite3.connect('tienda.db') as conn:
    query = '''
        SELECT 
            p.nombre,
            p.precio,
            COALESCE(v.cantidad, 0) as cantidad,
            COALESCE(p.precio * v.cantidad, 0) as total
        FROM productos p
        LEFT JOIN ventas v ON p.id = v.producto_id
    '''
    df = pd.read_sql(query, conn)
    
print("📊 LEFT JOIN - Todos los productos (con NULL si no hay ventas):")
print(df)

# Encontrar productos sin ventas
with sqlite3.connect('tienda.db') as conn:
    query = '''
        SELECT p.nombre, p.precio
        FROM productos p
        LEFT JOIN ventas v ON p.id = v.producto_id
        WHERE v.id IS NULL
    '''
    df_sin_ventas = pd.read_sql(query, conn)
    
print("\n❌ Productos sin ventas:")
print(df_sin_ventas)

JOINs Múltiples

En aplicaciones reales, frecuentemente necesitas combinar datos de 3 o más tablas. Esto es común cuando tienes relaciones many-to-many o datos normalizados.

Ejemplo: Sistema de Facturas con 3 Tablas

import sqlite3
import pandas as pd

# Crear base de datos con 3 tablas relacionadas
with sqlite3.connect('facturacion.db') as conn:
    cursor = conn.cursor()
    
    # Tabla de facturas
    cursor.execute('''
        CREATE TABLE IF NOT EXISTS facturas (
            id INTEGER PRIMARY KEY,
            cliente TEXT,
            pais TEXT
        )
    ''')
    
    # Tabla de items de factura
    cursor.execute('''
        CREATE TABLE IF NOT EXISTS items_factura (
            id INTEGER PRIMARY KEY,
            factura_id INTEGER,
            producto_id INTEGER,
            cantidad INTEGER,
            FOREIGN KEY (factura_id) REFERENCES facturas(id),
            FOREIGN KEY (producto_id) REFERENCES productos(id)
        )
    ''')
    
    # Tabla de productos
    cursor.execute('''
        CREATE TABLE IF NOT EXISTS productos (
            id INTEGER PRIMARY KEY,
            nombre TEXT,
            precio REAL
        )
    ''')
    
    # Insertar datos
    facturas = [(1, 'Ana García', 'España'), (2, 'John Smith', 'USA'), (3, 'Marie Dubois', 'Francia')]
    cursor.executemany('INSERT OR IGNORE INTO facturas VALUES (?,?,?)', facturas)
    
    productos = [(1, 'Laptop', 1000), (2, 'Mouse', 20), (3, 'Teclado', 50)]
    cursor.executemany('INSERT OR IGNORE INTO productos VALUES (?,?,?)', productos)
    
    items = [
        (1, 1, 1, 2),  # Factura 1: 2 Laptops
        (2, 1, 2, 1),  # Factura 1: 1 Mouse
        (3, 2, 3, 3),  # Factura 2: 3 Teclados
        (4, 3, 1, 1),  # Factura 3: 1 Laptop
    ]
    cursor.executemany('INSERT OR IGNORE INTO items_factura VALUES (?,?,?,?)', items)
    conn.commit()

# Consulta con JOIN múltiple
with sqlite3.connect('facturacion.db') as conn:
    query = '''
        SELECT 
            f.cliente,
            f.pais,
            p.nombre AS producto,
            i.cantidad,
            p.precio,
            (i.cantidad * p.precio) AS total
        FROM facturas f
        JOIN items_factura i ON f.id = i.factura_id
        JOIN productos p ON i.producto_id = p.id
        ORDER BY f.cliente, p.nombre
    '''
    df = pd.read_sql(query, conn)
    
print("📋 DETALLE DE FACTURAS (3 tablas combinadas):")
print(df)

# Resumen por país
with sqlite3.connect('facturacion.db') as conn:
    query = '''
        SELECT 
            f.pais,
            COUNT(DISTINCT f.id) AS num_facturas,
            SUM(i.cantidad * p.precio) AS ingresos_totales
        FROM facturas f
        JOIN items_factura i ON f.id = i.factura_id
        JOIN productos p ON i.producto_id = p.id
        GROUP BY f.pais
        ORDER BY ingresos_totales DESC
    '''
    df_pais = pd.read_sql(query, conn)
    
print("\n🌍 INGRESOS POR PAÍS:")
print(df_pais)

Agregaciones y Subconsultas

SQL ofrece potentes funciones de agregación y la capacidad de anidar consultas (subconsultas) para análisis complejos.

Funciones de Agregación Comunes

import sqlite3
import pandas as pd

# Usar la base de datos de empleados anterior
with sqlite3.connect('mi_empresa.db') as conn:
    # COUNT, AVG, MIN, MAX, SUM
    query = '''
        SELECT 
            departamento,
            COUNT(*) AS num_empleados,
            AVG(salario) AS salario_promedio,
            MIN(salario) AS salario_minimo,
            MAX(salario) AS salario_maximo,
            SUM(salario) AS masa_salarial
        FROM empleados
        GROUP BY departamento
        HAVING num_empleados > 1
        ORDER BY masa_salarial DESC
    '''
    df = pd.read_sql(query, conn)
    
print("📊 ANÁLISIS POR DEPARTAMENTO (con HAVING):")
print(df)
📊 ANÁLISIS POR DEPARTAMENTO (con HAVING): departamento num_empleados salario_promedio salario_minimo salario_maximo masa_salarial 0 IT 2 47500.00 45000 50000 95000 1 Ventas 2 47500.00 40000 55000 95000

Subconsultas (Subqueries)

Las subconsultas permiten usar el resultado de una consulta dentro de otra consulta. Son útiles para filtrar datos basándose en cálculos agregados.

1. Subconsulta en WHERE

import sqlite3
import pandas as pd

# Empleados con salario mayor al promedio
with sqlite3.connect('mi_empresa.db') as conn:
    query = '''
        SELECT nombre, salario, departamento
        FROM empleados
        WHERE salario > (SELECT AVG(salario) FROM empleados)
        ORDER BY salario DESC
    '''
    df = pd.read_sql(query, conn)
    
print("💰 Empleados con salario mayor al promedio:")
print(df)
💰 Empleados con salario mayor al promedio: nombre salario departamento 0 Pedro Sánchez 60000.0 Marketing 1 Juan López 55000.0 Ventas 2 María Rodríguez 50000.0 IT

2. Subconsulta en SELECT

import sqlite3
import pandas as pd

# Comparar cada empleado con el promedio de su departamento

    query = '''

            departamento,

        FROM empleados e1
        ORDER BY departamento, salario DESC
    '''

3. Subconsulta con IN

import sqlite3
import pandas as pd

# Empleados en departamentos con más de 1 empleado
with sqlite3.connect('mi_empresa.db') as conn:
    query = '''
        SELECT nombre, departamento, salario
        FROM empleados
        WHERE departamento IN (
            SELECT departamento
            FROM empleados
            GROUP BY departamento
            HAVING COUNT(*) > 1
        )
        ORDER BY departamento, salario DESC
    '''
    df = pd.read_sql(query, conn)
    
print("👥 Empleados en departamentos grandes (>1 empleado):")
print(df)

Common Table Expressions (CTE) con WITH

Los CTEs hacen las consultas complejas más legibles al permitirte definir "tablas temporales" que existen solo durante la ejecución de la consulta.

import sqlite3
import pandas as pd

with sqlite3.connect('mi_empresa.db') as conn:
    query = '''
        WITH estadisticas_dept AS (
            SELECT 
                departamento,
                AVG(salario) AS salario_promedio,
                COUNT(*) AS num_empleados
            FROM empleados
            GROUP BY departamento
        ),
        empleados_senior AS (
            SELECT 
                nombre,
                departamento,
                salario
            FROM empleados
            WHERE edad >= 30
        )
        SELECT 
            es.nombre,
            es.departamento,
            es.salario,
            sd.salario_promedio,
            sd.num_empleados,
            es.salario - sd.salario_promedio AS diferencia_promedio
        FROM empleados_senior es
        JOIN estadisticas_dept sd ON es.departamento = sd.departamento
        ORDER BY diferencia_promedio DESC
    '''
    df = pd.read_sql(query, conn)
    
print("🎯 Análisis con CTE (empleados senior vs promedio dept):")
print(df.to_string(index=False))

Aplicación Real con SQL y Pandas

import sqlite3
import pandas as pd
from datetime import datetime, timedelta
import random

# 1. Crear base de datos y tablas
with sqlite3.connect('ventas.db') as conn:
    cursor = conn.cursor()
    
    # Tabla de productos
    cursor.execute('''
        CREATE TABLE IF NOT EXISTS productos (
            id INTEGER PRIMARY KEY,
            nombre TEXT NOT NULL,
            precio REAL NOT NULL
        )
    ''')
    
    # Tabla de vendedores
    cursor.execute('''
        CREATE TABLE IF NOT EXISTS vendedores (
            id INTEGER PRIMARY KEY,
            nombre TEXT NOT NULL
        )
    ''')
    
    # Tabla de ventas
    cursor.execute('''
        CREATE TABLE IF NOT EXISTS ventas (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            producto_id INTEGER,
            vendedor_id INTEGER,
            cantidad INTEGER,
            fecha DATE,
            FOREIGN KEY (producto_id) REFERENCES productos(id),
            FOREIGN KEY (vendedor_id) REFERENCES vendedores(id)
        )
    ''')

# 2. Insertar datos de ejemplo
with sqlite3.connect('ventas.db') as conn:
    cursor = conn.cursor()
    
    # Productos
    productos = [
        (1, 'Laptop', 1000),
        (2, 'Mouse', 20),
        (3, 'Teclado', 50),
        (4, 'Monitor', 300)
    ]
    cursor.executemany('INSERT OR IGNORE INTO productos VALUES (?,?,?)', productos)
    
    # Vendedores
    vendedores = [
        (1, 'Ana'), (2, 'Juan'), (3, 'María'), (4, 'Pedro')
    ]
    cursor.executemany('INSERT OR IGNORE INTO vendedores VALUES (?,?)', vendedores)
    
    # Generar ventas aleatorias
    ventas = []
    fecha_inicio = datetime.now() - timedelta(days=30)
    for _ in range(100):
        venta = (
            random.randint(1, 4),  # producto_id
            random.randint(1, 4),  # vendedor_id
            random.randint(1, 10), # cantidad
            (fecha_inicio + timedelta(days=random.randint(0, 30))).date()
        )
        ventas.append(venta)
    
    cursor.executemany('''
        INSERT INTO ventas (producto_id, vendedor_id, cantidad, fecha)
        VALUES (?,?,?,?)
    ''', ventas)
    conn.commit()

# 3. Análisis con SQL y Pandas

with sqlite3.connect('ventas.db') as conn:
    # Top productos
    query_productos = '''
        SELECT 
            p.nombre,
            SUM(v.cantidad) as unidades_vendidas,
            SUM(v.cantidad * p.precio) as ingresos_totales
        FROM ventas v
        JOIN productos p ON v.producto_id = p.id
        GROUP BY p.nombre
        ORDER BY ingresos_totales DESC
    '''
    df_productos = pd.read_sql(query_productos, conn)
    
    print("\n1️⃣ TOP PRODUCTOS POR INGRESOS:")
    print(df_productos.to_string(index=False))
    
    # Top vendedores
    query_vendedores = '''
        SELECT 
            vd.nombre,
            COUNT(*) as num_ventas,
            SUM(v.cantidad * p.precio) as ingresos_totales
        FROM ventas v
        JOIN vendedores vd ON v.vendedor_id = vd.id
        JOIN productos p ON v.producto_id = p.id
        GROUP BY vd.nombre
        ORDER BY ingresos_totales DESC
    '''
    df_vendedores = pd.read_sql(query_vendedores, conn)
    
    print("\n2️⃣ RENDIMIENTO DE VENDEDORES:")
    print(df_vendedores.to_string(index=False))
    
    # Tendencia semanal

        SELECT 
            strftime('%Y-%W', fecha) as semana,
            COUNT(*) as num_ventas,
            SUM(v.cantidad * p.precio) as ingresos
        FROM ventas v
        JOIN productos p ON v.producto_id = p.id
        GROUP BY semana
        ORDER BY semana
    '''
    df_tendencia = pd.read_sql(query_tendencia, conn)
    
    print("\n3️⃣ TENDENCIA SEMANAL:")
    print(df_tendencia.to_string(index=False))

📚 Cheat Sheet de SQL con Python

Conexión y Setup

import sqlite3 Importar módulo SQLite
conn = sqlite3.connect('db.db') Conectar a base de datos
cursor = conn.cursor() Crear cursor para ejecutar SQL
conn.commit() Guardar cambios
conn.close() Cerrar conexión

Operaciones CRUD

CREATE TABLE tabla (...) Crear tabla
INSERT INTO tabla VALUES (?...) Insertar datos (usar ?)
SELECT * FROM tabla Seleccionar datos
UPDATE tabla SET col=? WHERE ... Actualizar registros
DELETE FROM tabla WHERE ... Eliminar registros

Consultas y Filtros

SELECT col1, col2 FROM tabla Seleccionar columnas específicas
WHERE condicion Filtrar resultados
ORDER BY col ASC/DESC Ordenar resultados
LIMIT n Limitar número de resultados
WHERE col LIKE '%texto%' Búsqueda de patrones

JOINs

INNER JOIN tabla2 ON t1.id=t2.id Solo coincidencias en ambas tablas
LEFT JOIN tabla2 ON t1.id=t2.id Todas de izquierda + coincidencias
WHERE t2.id IS NULL Encontrar filas sin coincidencia

Agregaciones

COUNT(*) Contar filas
SUM(col) Suma de valores
AVG(col) Promedio
MIN(col) / MAX(col) Mínimo y máximo
GROUP BY col Agrupar para agregación
HAVING condicion Filtrar grupos (después GROUP BY)

Integración con Pandas

pd.read_sql(query, conn) Leer consulta SQL a DataFrame
pd.read_sql_query(query, conn) Alias de read_sql
df.to_sql('tabla', conn) Guardar DataFrame en SQL
if_exists='append' Agregar datos a tabla existente
if_exists='replace' Reemplazar tabla completa

SQLAlchemy

from sqlalchemy import create_engine Importar SQLAlchemy
create_engine('sqlite:///db.db') Crear engine SQLite
create_engine('postgresql://user:pwd@host/db') Conectar a PostgreSQL
pd.read_sql(query, engine) Usar engine con Pandas

Métodos del Cursor

cursor.execute(query) Ejecutar consulta SQL
cursor.execute(query, (val,)) Ejecutar con parámetros seguros
cursor.executemany(query, data) Ejecutar múltiples inserciones
cursor.fetchone() Obtener una fila
cursor.fetchall() Obtener todas las filas
cursor.fetchmany(n) Obtener n filas
