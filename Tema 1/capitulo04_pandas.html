Introducción a Pandas

Mientras que NumPy se especializa en arrays homogéneos de datos numéricos, Pandas proporciona estructuras para manipular datos heterogéneos organizados en formato tabular: nombres, fechas, categorías y números, tal como se presentan en aplicaciones reales.

Pandas fue desarrollado por Wes McKinney en 2008 durante su trabajo en AQR Capital Management. La necesidad de herramientas más eficientes para manejar series temporales financieras y datos tabulares complejos motivó su creación. Actualmente es la biblioteca de análisis de datos más utilizada en Python.

Pandas en el Contexto del Análisis de Datos

Pandas como herramienta de análisis exploratorio

NumPy proporciona eficientes operaciones matemáticas con arrays homogéneos. Sin embargo, en aplicaciones reales los datos poseen características que van más allá de matrices numéricas:

  • Tipos heterogéneos: números, texto, fechas en la misma tabla
  • Etiquetas significativas: nombres de columnas, identificadores de fila
  • Valores incompletos: datos que requieren decisiones sobre su tratamiento
  • Estructura tabular: filas y columnas con relaciones semánticas

Pandas resuelve estos requerimientos proporcionando estructuras y métodos diseñados específicamente para análisis exploratorio de datos. Su interfaz combina características de hojas de cálculo (como Excel) con la flexibilidad de SQL, integrando ambas dentro del ecosistema Python.

Comparación de herramientas: Excel, SQL y Pandas

Escenario Excel SQL Pandas
Volumen de datos < 100k filas Ilimitado < 2GB RAM
Velocidad Lenta Rápida Muy rápida
Automatización Difícil Fácil Muy fácil
Análisis exploratorio Adecuado Limitado Excelente
Gestión de datos incompletos Manual Prevención Herramientas integradas

Flujo de trabajo típico

El análisis con Pandas generalmente sigue un patrón predecible:

  • Carga: Importar datos desde archivos o bases de datos
  • Inspección: Examinar estructura, tipos y completitud de los datos
  • Limpieza: Resolver valores faltantes e inconsistencias
  • Transformación: Crear columnas nuevas, combinar datos, reorganizar
  • Análisis: Calcular estadísticas, agrupaciones, tendencias
  • Exportación: Guardar resultados y reportes

Estructuras Fundamentales: Series y DataFrames

Las Dos Estructuras de Datos en Pandas

Todo en Pandas se construye sobre dos estructuras. Aprenderlas bien es fundamental:

Estructura Dimensiones Analogía Cuándo la usas
Series 1D (una columna) Columna de Excel Datos de una variable única
DataFrame 2D (múltiples columnas) Hoja de Excel completa Datos tabulares con múltiples variables

Series: Estructura Unidimensional con Índices

Una Series es un array unidimensional que asocia cada valor con una etiqueta de índice. A diferencia de un array de NumPy que accede por posición numérica, una Series permite acceso por etiqueta, proporcionando mayor claridad y seguridad.

Anatomía de una Series

Internamente, una Series contiene dos componentes alineados:

Índice:  Lun    Mar    Mié    Jue    Vie
Valor:   22     24     19     25     23
Tipo:    int64  int64  int64  int64  int64

El índice actúa como un "diccionario" que mapea cada posición a una etiqueta. Esto es fundamental para operaciones entre Series, que se alinean automáticamente por índice en lugar de por posición.

Crear Series

Opción 1: Desde una Lista (Índice Automático)

import pandas as pd

# Crea una Series con índice numérico automático (0, 1, 2, ...)
temperaturas = pd.Series([22, 24, 19, 25, 23])
print(temperaturas)

# Acceso por posición (igual que arrays)
print(f"Primera temperatura: {temperaturas[0]}°C")
print(f"Última temperatura: {temperaturas[-1]}°C")
0 22 1 24 2 19 3 25 4 23 dtype: int64 Primera temperatura: 22°C Última temperatura: 23°C

Opción 2: Con Índices Personalizados (Lo Importante)

# Índices personalizados: nombres de días
dias = ['Lun', 'Mar', 'Mié', 'Jue', 'Vie']
temperaturas = pd.Series([22, 24, 19, 25, 23], index=dias)
print(temperaturas)
print()

# Acceso por etiqueta (la verdadera potencia)
print(f"Temperatura del miércoles: {temperaturas['Mié']}°C")
print(f"Temperaturas de los primeros 3 días:")
print(temperaturas[['Lun', 'Mar', 'Mié']])
Lun 22 Mar 24 Mié 19 Jue 25 Vie 23 dtype: int64 Temperatura del miércoles: 19°C Temperaturas de los primeros 3 días: Lun 22 Mar 24 Mié 19 dtype: int64

Opción 3: Desde un Diccionario (Las Etiquetas Salen Automáticamente)

# Diccionario: clave → índice, valor → valor de la Series
poblacion = pd.Series({
    'Madrid': 3_223_334,
    'Barcelona': 1_620_343,
    'Valencia': 791_413,
    'Sevilla': 688_592
})

print(poblacion)
print()

# Acceso inmediato por ciudad
print(f"Población de Barcelona: {poblacion['Barcelona']:,}")
print(f"Ciudad más poblada: {poblacion.idxmax()}")
print(f"Total población: {poblacion.sum():,}")
Madrid 3223334 Barcelona 1620343 Valencia 791413 Sevilla 688592 dtype: int64 Población de Barcelona: 1,620,343 Ciudad más poblada: Madrid Total población: 6,323,682

Operaciones Aritméticas: Alineación por Índice

Una ventaja fundamental de las Series respecto a arrays de NumPy es que las operaciones aritméticas se alinean automáticamente por índice en lugar de por posición. Esto evita errores cuando el orden de los datos cambia.

# Ventas de dos meses (mismos productos)
enero = pd.Series([100, 150, 200], index=['Laptop', 'Mouse', 'Teclado'])
febrero = pd.Series([120, 160, 180], index=['Laptop', 'Mouse', 'Teclado'])

print("Enero:")
print(enero)
print("\nFebrero:")
print(febrero)

# Resta alineada: Laptop - Laptop, Mouse - Mouse, etc.
crecimiento = febrero - enero
print("\nCrecimiento (unidades):")
print(crecimiento)

# Cálculo de porcentajes
pct_crecimiento = ((febrero - enero) / enero * 100).round(1)
print("\nCrecimiento (%):")
print(pct_crecimiento)
Enero: Laptop 100 Mouse 150 Teclado 200 dtype: int64 Febrero: Laptop 120 Mouse 160 Teclado 180 dtype: int64 Crecimiento (unidades): Laptop 20 Mouse 10 Teclado -20 dtype: int64 Crecimiento (%): Laptop 20.0 Mouse 6.7 Teclado -10.0 dtype: float64

DataFrames: Estructura Tabular

Definición y estructura

Un DataFrame es una estructura bidimensional tabular con filas y columnas etiquetadas. Puede considerarse como un diccionario de Series que comparten el mismo índice (filas).

Internamente:

                      Laptop    Mouse    Teclado
Ana                   100        50         30
Juan                  120        60         35
María                 110        55         32

Cada columna es una Series
Todas comparten el mismo índice (filas)

Creación de DataFrames

Desde diccionario de listas

Estructura de datos más común: las claves se convierten en nombres de columnas.

import pandas as pd

datos = {
    'nombre': ['Ana', 'Juan', 'María', 'Pedro'],

    'salario': [30000, 45000, 55000, 38000],
    'departamento': ['Ventas', 'IT', 'IT', 'Ventas']

df = pd.DataFrame(datos)
print(df)

Estructura común cuando los datos provienen de APIs o ficheros semiestructurados.

Selección e Indexación de Datos

La capacidad de extraer subconjuntos específicos de datos es fundamental en análisis de datos. Pandas proporciona múltiples métodos adaptados a diferentes situaciones.

Selección de columnas

La forma más básica de seleccionar datos es acceder a columnas completas:

# Crear DataFrame de ejemplo
empleados = pd.DataFrame({
    'nombre': ['Ana', 'Juan', 'María', 'Pedro', 'Laura'],
    'edad': [25, 30, 35, 28, 32],
    'salario': [30000, 45000, 55000, 38000, 48000],
    'departamento': ['Ventas', 'IT', 'IT', 'Ventas', 'Marketing']
})

# Una columna: devuelve Series
nombres = empleados['nombre']
print("Tipo:", type(nombres))
print(nombres)
print()

# Múltiples columnas: devuelve DataFrame
info_basica = empleados[['nombre', 'edad']]
print("Tipo:", type(info_basica))
print(info_basica)
Tipo: 0 Ana 1 Juan 2 María 3 Pedro 4 Laura Name: nombre, dtype: object Tipo: nombre edad 0 Ana 25 1 Juan 30 2 María 35 3 Pedro 28 4 Laura 32

Filtrado con condiciones booleanas

Para seleccionar filas que cumplen condiciones específicas se utilizan arrays booleanos:

# ¿Quiénes ganan más de 40,000?
salarios_altos = empleados[empleados['salario'] > 40000]
print(salarios_altos)

# Desglosado (lo que ocurre internamente):
mascara = empleados['salario'] > 40000  # Array de True/False
print("\nMáscara:", mascara.values)
resultado = empleados[mascara]  # Pandas filtra por True
nombre edad salario departamento 1 Juan 30 45000 IT 2 María 35 55000 IT 4 Laura 32 48000 Marketing Máscara: [False True True False True]

Condiciones múltiples

Generalmente es necesario combinar varias condiciones con operadores lógicos:

AND: Ambas Condiciones (& operador)

# Pregunta: Jóvenes (< 30) que ganan bien (> 40,000)
jovenes_bien_pagados = empleados[
    (empleados['edad'] < 30) & (empleados['salario'] > 40000)
]
print("Jóvenes (< 30) bien pagados (> 40k):")
print(jovenes_bien_pagados)
nombre edad salario departamento 1 Juan 30 45000 IT

OR: Al Menos Una Condición (| operador)

# Pregunta: Personas en IT O Marketing
it_o_marketing = empleados[
    (empleados['departamento'] == 'IT') | 
    (empleados['departamento'] == 'Marketing')
]
print("IT o Marketing:")
print(it_o_marketing)

# Alternativa más clara con .isin()
it_o_marketing_v2 = empleados[empleados['departamento'].isin(['IT', 'Marketing'])]
print("\nMismo resultado con .isin():")
print(it_o_marketing_v2)

NOT: Negación (~ operador)

# Pregunta: TODO EXCEPTO Ventas
no_ventas = empleados[~(empleados['departamento'] == 'Ventas')]
print("No Ventas:")
print(no_ventas)

Indexación avanzada: loc e iloc

Para mayor control sobre la selección, Pandas proporciona los métodos loc (selección por etiqueta) e iloc (selección por posición).

loc: selección por etiqueta

loc selecciona datos utilizando nombres de índice o columnas. Es especialmente útil en análisis exploratorio donde la claridad del código es importante.

# Crear DataFrame con índice personalizado
empleados_indexado = empleados.set_index('nombre')
print(empleados_indexado)
print()

# Fila específica por etiqueta
print("Datos de Juan:")
print(empleados_indexado.loc['Juan'])
print()

# Rango de filas (incluye el final, A DIFERENCIA de slicing normal)
print("De Ana a María (inclusivo):")
print(empleados_indexado.loc['Ana':'María'])
print()

# Seleccionar columnas específicas
print("Edad y salario de personas de IT:")
print(empleados.loc[empleados['departamento'] == 'IT', ['edad', 'salario']])
edad salario departamento nombre Ana 25 30000 Ventas Juan 30 45000 IT María 35 55000 IT Pedro 28 38000 Ventas Laura 32 48000 Marketing Datos de Juan: edad 30 salario 45000 departamento IT Name: Juan, dtype: object De Ana a María (inclusivo): edad salario departamento nombre Ana 25 30000 Ventas Juan 30 45000 IT María 35 55000 IT Edad y salario de personas de IT: edad salario 1 30 45000 2 35 55000

iloc: selección por posición

iloc selecciona datos por posición numérica (0, 1, 2, ...). Es útil en scripts donde es necesario acceder a "las primeras N filas" o "las últimas M columnas".

# Primera fila
print("Primera fila (índice 0):")
print(empleados.iloc[0])
print()

# Primeras 3 filas, primeras 2 columnas
print("Primeras 3 filas, primeras 2 columnas:")
print(empleados.iloc[:3, :2])
print()

# Última fila
print("Última fila:")
print(empleados.iloc[-1])

Manejo de Valores Faltantes

En datos reales siempre existen valores incompletos. Las decisiones sobre cómo tratarlos afectan significativamente la calidad del análisis posterior.

Detección de valores faltantes

import pandas as pd
import numpy as np

# Datos incompletos (como en la realidad)
datos = {
    'nombre': ['Ana', 'Juan', 'María', 'Pedro', 'Laura'],
    'edad': [25, np.nan, 35, 28, 32],
    'salario': [30000, 45000, np.nan, 38000, 48000],
    'email': ['ana@email.com', None, 'maria@email.com', 'pedro@email.com', None]
}
df = pd.DataFrame(datos)

# ¿Dónde están los nulos?
print("Conteo de valores faltantes por columna:")
nulos_por_columna = df.isnull().sum()
print(nulos_por_columna)
print()

# Porcentaje (más importante que conteo)
print("Porcentaje de datos faltantes:")
porcentaje = (df.isnull().sum() / len(df) * 100).round(1)
print(porcentaje)
print()

# ¿Cuáles son las filas con problemas?
print("Filas con algún valor faltante:")
print(df[df.isnull().any(axis=1)])
Conteo de valores faltantes por columna: nombre 0 edad 1 salario 1 email 2 dtype: int64 Porcentaje de datos faltantes: nombre 0.0 edad 20.0 salario 20.0 email 40.0 dtype: float64 Filas con algún valor faltante: nombre edad salario email 1 Juan NaN 45000.0 None 2 María 35.0 NaN maria@email.com 4 Laura 32.0 48000.0 None

Estrategias de tratamiento

Existen tres enfoques principales para resolver valores faltantes, cada uno apropiado en diferentes contextos:

Eliminación de filas

Apropiado cuando el conjunto de datos es suficientemente grande y los valores faltantes son pocos y distribuidos aleatoriamente.

# Opción 1: Eliminar filas con CUALQUIER nulo
sin_nulos = df.dropna()
print("Después de .dropna():")
print(sin_nulos)
print(f"Filas originales: {len(df)}, Filas restantes: {len(sin_nulos)}")
print()

# Opción 2: Eliminar solo si falta la columna crítica
sin_nulos_salario = df[df['salario'].notna()]
print("Sin valores faltantes en salario:")
print(sin_nulos_salario)
Después de .dropna(): nombre edad salario email 0 Ana 25.0 30000.0 ana@email.com 3 Pedro 28.0 38000.0 pedro@email.com Filas originales: 5, Filas restantes: 2 Sin valores faltantes en salario: nombre edad salario email 0 Ana 25.0 30000.0 ana@email.com 1 Juan NaN 45000.0 None 3 Pedro 28.0 38000.0 pedro@email.com 4 Laura 32.0 48000.0 None

Imputación estadística

Consiste en reemplazar valores faltantes con estadísticos calculados (media, mediana) del conjunto de datos. Conserva el tamaño del dataset.

# Rellenar edad con la mediana
mediana_edad = df['edad'].median()
df['edad'].fillna(mediana_edad, inplace=True)
print(f"Edad faltante rellenada con mediana: {mediana_edad}")

# Rellenar salario con la media
media_salario = df['salario'].mean()
df['salario'].fillna(media_salario, inplace=True)
print(f"Salario faltante rellenado con media: {media_salario:.0f}")

print("\nDataFrame después de rellenar:")
print(df)

Imputación basada en contexto

Reemplazar valores faltantes con valores específicos que tienen sentido en el dominio del problema.

# Rellenar email faltante con valor especial
df['email'].fillna('sin_contacto@empresa.com', inplace=True)

print("Después de rellenar email:")
print(df)

Agrupación y Agregación con GroupBy

groupby() implementa el patrón "Dividir-Aplicar-Combinar": divide los datos en grupos según una o más columnas, aplica una operación a cada grupo, y combina los resultados.

Agrupación simple

import pandas as pd

# Dataset de ventas
ventas = pd.DataFrame({
    'producto': ['Laptop', 'Mouse', 'Teclado', 'Laptop', 'Mouse', 'Teclado'],
    'vendedor': ['Ana', 'Ana', 'Ana', 'Juan', 'Juan', 'Juan'],
    'unidades': [2, 10, 5, 3, 8, 6],
    'precio': [1000, 20, 50, 1000, 20, 50]
})

print("Datos originales:")
print(ventas)
print()

# Pregunta: ¿Cuántas unidades por producto?
unidades_por_producto = ventas.groupby('producto')['unidades'].sum()
print("Unidades vendidas por producto:")
print(unidades_por_producto)
print()

# Esto es equivalente a:
# 1. Dividir por producto: Laptop {}, Mouse {}, Teclado {}
# 2. Aplicar sum() a cada grupo
# 3. Combinar resultados en una Series
Datos originales: producto vendedor unidades precio 0 Laptop Ana 2 1000 1 Mouse Ana 10 20 2 Teclado Ana 5 50 3 Laptop Juan 3 1000 4 Mouse Juan 8 20 5 Teclado Juan 6 50 Unidades vendidas por producto: producto Laptop 5 Mouse 18 Teclado 11 Name: unidades, dtype: int64

Funciones de Agregación Comunes

# Diferentes funciones de agregación
resumen = ventas.groupby('producto')['unidades'].agg({
    'sum': 'sum',           # Total
    'count': 'count',       # Cuántas veces aparece
    'mean': 'mean',         # Promedio
    'max': 'max',           # Máximo
    'min': 'min'            # Mínimo
})

print("Resumen de unidades por producto:")
print(resumen)
Resumen de unidades por producto: sum count mean max min producto Laptop 5 2 2.5 3 2 Mouse 18 2 9.0 10 8 Teclado 11 2 5.5 6 5

Agrupación por múltiples columnas

# Agrupar por MÚLTIPLES columnas
por_vendedor_producto = ventas.groupby(['vendedor', 'producto'])['unidades'].sum()
print("Unidades por vendedor y producto:")
print(por_vendedor_producto)
print()

# Agregar columna de total de venta
ventas['venta_total'] = ventas['unidades'] * ventas['precio']

# Múltiples agregaciones a la vez
resumen_completo = ventas.groupby('vendedor').agg({
    'venta_total': 'sum',    # Total facturado
    'unidades': 'sum',       # Total unidades
    'precio': 'mean'         # Precio promedio
})

print("Resumen por vendedor:")
print(resumen_completo)

Operaciones avanzadas

Más allá de las agregaciones simples, existen métodos para operaciones complejas sobre grupos:

transform(): retorna tabla con misma forma que la entrada

A diferencia de agg(), transform() retorna un resultado con el mismo número de filas que los datos originales.

empleados = pd.DataFrame({
    'departamento': ['Ventas', 'Ventas', 'IT', 'IT', 'IT', 'RRHH'],
    'nombre': ['Ana', 'Juan', 'María', 'Pedro', 'Laura', 'Carlos'],
    'salario': [35000, 40000, 55000, 60000, 52000, 38000]
})

empleados['salario_medio_dept'] = empleados.groupby('departamento')['salario'].transform('mean')
empleados['diff_vs_media'] = empleados['salario'] - empleados['salario_medio_dept']

print(empleados[['nombre', 'salario', 'salario_medio_dept', 'diff_vs_media']])

apply(): aplicar función personalizada a cada grupo

Permite ejecutar lógica personalizada en cada grupo.

# Función personalizada para procesar cada grupo
def top_n_salarios(group, n=2):
    """Devuelve los n salarios más altos del grupo"""
    return group.nlargest(n, 'salario')

# apply() ejecuta la función en cada grupo y combina
print("Top 2 salarios por departamento:")
top_salarios = empleados.groupby('departamento', group_keys=False).apply(
    lambda x: top_n_salarios(x, n=2)
)
print(top_salarios)

Combinación de DataFrames

Cuando los datos se distribuyen en múltiples tablas, existen dos enfoques fundamentales para combinarlos: concatenación (apilar) y merge (combinar por columna común).

Concatenación

Apila DataFrames verticalmente (añadiendo filas). Apropiado cuando las estructuras son idénticas pero los datos provienen de diferentes períodos o fuentes.

# Ventas de dos trimestres (misma estructura)

Merging/Joining: combinación relacional

Combina DataFrames basándose en una columna común, similar a operaciones JOIN en SQL. Existen cuatro tipos de joins con diferentes comportamientos respecto a registros sin coincidencias.

Tipo Comportamiento Descripción
how='inner' Solo coincidencias exactas en ambas tablas Retorna solo filas donde existe clave en ambas tablas
how='left' Todas las filas de la izquierda más coincidencias Mantiene todos los registros de la tabla izquierda
how='right' Coincidencias más todas las filas de la derecha Mantiene todos los registros de la tabla derecha
how='outer' Unión de ambas tablas Retorna todas las filas de ambas tablas con NaN donde no coincide

Inner Join

import pandas as pd

# Tabla 1: Empleados
empleados = pd.DataFrame({
    'emp_id': [1, 2, 3, 4],
    'nombre': ['Ana', 'Juan', 'María', 'Pedro'],
    'dept_id': [10, 20, 20, 30]

Transformación de Datos: Pivot Tables y Reshaping

Los datos pueden necesitar ser reorganizados pasando entre formato "largo" (una fila por observación) y formato "ancho" (categorías como columnas). Las operaciones de pivoting permiten realizar estas transformaciones.

Formatos largo y ancho

Todo conjunto de datos puede representarse de dos formas:

  • Formato largo ("Long"): Una fila por observación. Facilita análisis y cálculos.
  • Formato ancho ("Wide"): Categorías como columnas. Facilita presentación y comparación visual.
Formato Característica Cuando se usa
Largo (Long) Una fila por observación Análisis y cálculos
Ancho (Wide) Categorías como columnas Presentación y comparación visual

pivot_table(): convertir de largo a ancho

Transforma datos del formato largo al ancho, agregando múltiples observaciones por grupo en una sola fila.

import pandas as pd

# Datos de ventas (formato largo)
ventas = pd.DataFrame({
    'mes': ['Ene', 'Ene', 'Ene', 'Feb', 'Feb', 'Feb', 'Mar', 'Mar', 'Mar'],
    'producto': ['Laptop', 'Mouse', 'Teclado'] * 3,
    'región': ['Norte', 'Norte', 'Norte', 'Sur', 'Sur', 'Sur', 'Norte', 'Norte', 'Norte'],
    'ventas': [100, 50, 30, 120, 60, 35, 110, 55, 32]
})

print("ANTES - Formato Largo (cada venta es una fila):")
print(ventas)
print()

# pivot_table: agrupa y resume
tabla_dinamica = ventas.pivot_table(
    values='ventas',        # Columna a resumir
    index='producto',       # Filas: productos
    columns='mes',          # Columnas: meses
    aggfunc='sum',          # Función: suma
    fill_value=0            # Reemplazar NaN con 0
)

print("DESPUÉS - Formato Ancho (una fila por producto):")
print(tabla_dinamica)
print()

# Usar múltiples índices
tabla_compleja = ventas.pivot_table(
    values='ventas',
    index=['región', 'producto'],  # Jerarquía: región > producto
    columns='mes',
    aggfunc='sum',
    fill_value=0,
    margins=True,           # Añade totales
    margins_name='TOTAL'
)

print("Con múltiples índices y totales:")
print(tabla_compleja)

melt(): convertir de ancho a largo

Realiza la operación inversa: convierte datos del formato ancho al largo, creando una fila por combinación de identificadores y valores.

# Datos en formato ancho (como Excel)
ventas_ancho = pd.DataFrame({
    'producto': ['Laptop', 'Mouse', 'Teclado'],
    'Ene': [100, 50, 30],
    'Feb': [120, 60, 35],
    'Mar': [110, 55, 32]
})

print("ANTES - Formato Ancho (meses como columnas):")
print(ventas_ancho)
print()

# melt: convertir ancho → largo
ventas_largo = ventas_ancho.melt(
    id_vars=['producto'],    # Columnas que se mantienen
    var_name='mes',          # Nombre para los meses
    value_name='ventas'      # Nombre para los valores
)

print("DESPUÉS - Formato Largo (una fila por venta):")
print(ventas_largo)

Resumen de métodos de transformación

  • pivot_table(): Transforma formato largo a ancho con agregación
  • melt(): Transforma formato ancho a largo
  • pivot(): Transforma formato largo a ancho sin agregación
  • stack/unstack(): Manipulan índices jerárquicos

Ejemplo Integral: Análisis de Ventas

El siguiente ejemplo muestra cómo utilizar los principales métodos de Pandas en un análisis completo de datos de ventas:

import pandas as pd
import numpy as np

# Crear datos de ventas simulados
np.random.seed(42)
fechas = pd.date_range('2024-01-01', periods=100, freq='D')

ventas = pd.DataFrame({
    'fecha': np.random.choice(fechas, 500),
    'producto': np.random.choice(['Laptop', 'Mouse', 'Teclado', 'Monitor'], 500),
    'vendedor': np.random.choice(['Ana', 'Juan', 'María', 'Pedro'], 500),
    'unidades': np.random.randint(1, 20, 500),
    'precio_unitario': np.random.choice([1000, 20, 50, 300], 500)
})

ventas['total'] = ventas['unidades'] * ventas['precio_unitario']

print("ANÁLISIS DE VENTAS")
print("=" * 60)

# 1. Resumen general
print("\n1. RESUMEN GENERAL")
print(f"Total de transacciones: {len(ventas):,}")
print(f"Venta total: ${ventas['total'].sum():,}")
print(f"Venta promedio: ${ventas['total'].mean():.2f}")

# 2. Top productos
print("\n2. PRODUCTOS CON MAYORES VENTAS")
top_productos = ventas.groupby('producto')['total'].sum().sort_values(ascending=False)
for producto, total in top_productos.items():
    print(f"   {producto:15} - ${total:,}")

# 3. Mejor vendedor
print("\n3. RENDIMIENTO POR VENDEDOR")
por_vendedor = ventas.groupby('vendedor').agg({
    'total': 'sum',
    'unidades': 'sum'
}).sort_values('total', ascending=False)
print(por_vendedor)

# 4. Ventas por mes
print("\n4. TENDENCIA MENSUAL")
ventas['mes'] = pd.to_datetime(ventas['fecha']).dt.to_period('M')
por_mes = ventas.groupby('mes')['total'].sum()
print(por_mes)

# 5. Producto más vendido por vendedor
print("\n5. PRODUCTOS POR VENDEDOR")
producto_vendedor = ventas.groupby(['vendedor', 'producto'])['unidades'].sum()
producto_vendedor = producto_vendedor.groupby('vendedor').idxmax()
for vendedor, (_, producto) in producto_vendedor.items():
    print(f"   {vendedor}: {producto}")

print("\n" + "=" * 60)
